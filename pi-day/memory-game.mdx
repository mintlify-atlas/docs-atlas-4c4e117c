title: "Pi memory game"
description: "An interactive keyboard-driven game that challenges players to type the decimal digits of π from memory, with three difficulty modes, audio feedback, and confetti on win."
---

The **Memory Game** section (`id="game"`) challenges players to type the digits of π after the decimal point (`1`, `4`, `1`, `5`, `9`, ...) one at a time. A hidden `<input>` element captures every keystroke so the game area retains focus and the cursor never leaves the display.

## Game modes

<Tabs>
  <Tab title="Normal">
    **3 lives.** Reach 50 correct digits to win. Each incorrect keystroke costs one life. Lose all three lives and the game ends.

    - Lives: 3
    - Win condition: 50 consecutive correct digits entered
    - Penalty: −1 life per error
    - Score: +10 points per correct digit
  </Tab>
  <Tab title="Contra Reloj (Time Attack)">
    **30 seconds.** Score as many digits as possible before time runs out. Errors do **not** deduct lives — the only limit is the clock.

    - Lives: 3 (displayed but not decremented on error)
    - Win condition: none — the game ends when the 30-second timer reaches zero
    - Penalty: none
    - Score: +10 points per correct digit
    - A pulsing red timer appears when fewer than 10 seconds remain
  </Tab>
  <Tab title="Muerte Súbita (Sudden Death)">
    **1 life. Infinite digits.** A single wrong digit ends the game immediately. There is no win condition — the goal is to set a personal record.

    - Lives: 1
    - Win condition: none (endless)
    - Penalty: game over on first error
    - Score: +10 points per correct digit
  </Tab>
</Tabs>

## Scoring

Every correctly typed digit awards **10 points**. The score is accumulated in a `score` state variable and displayed in the HUD above the digit display. There is no penalty to the score for errors — only lives (or time) are affected.

## Input capture

The game uses a hidden `<input>` element positioned absolutely behind the current digit cursor. It is always focused (auto-refocusing on blur) so the player can type freely without clicking:

```tsx
<input
  ref={inputRef}
  className="absolute inset-0 opacity-0 cursor-default"
  autoFocus
  onKeyDown={handleInput}
  onChange={(e) => { e.target.value = ''; }} // prevent value accumulation
  onBlur={() => inputRef.current?.focus()}   // keep focus at all times
/>
```

Only digit keys (`/^[0-9]$/`) are processed. All other keys are ignored.

## Audio feedback

All sounds are generated by the Web Audio API via `src/lib/audio.ts` — no audio files are loaded.

| Event | Function | Sound |
|-------|----------|-------|
| Correct digit | `playTypingBeep()` | Short square-wave burst at 600–800 Hz (randomized) |
| Wrong digit | `playErrorSound()` | Two descending sawtooth tones at 300 Hz and 250 Hz |
| Game won | `playSuccessSound()` | Ascending sine tones at 880 Hz then 1760 Hz |

```typescript
export const playTypingBeep = () => {
  playTone(600 + Math.random() * 200, 'square', 0.05, 0.02);
};

export const playErrorSound = () => {
  playTone(300, 'sawtooth', 0.3, 0.2);
  setTimeout(() => playTone(250, 'sawtooth', 0.4, 0.2), 100);
};

export const playSuccessSound = () => {
  playTone(880, 'sine', 0.1, 0.1);
  setTimeout(() => playTone(1760, 'sine', 0.2, 0.1), 100);
};
```

## Visual flash feedback

The outer `glass-panel` card changes border color and box-shadow on every keypress:

- **Correct digit:** green border + `box-shadow: 0 0 30px rgba(0,255,0,0.3)` for 200 ms
- **Incorrect digit:** red border + `box-shadow: 0 0 30px rgba(255,0,0,0.3)` + dark red background tint for 400 ms

```tsx
<div className={cn(
  "glass-panel rounded-3xl p-8 transition-colors duration-300",
  flash === 'success' ? "border-green-500 shadow-[0_0_30px_rgba(0,255,0,0.3)]" : "",
  flash === 'error'   ? "border-red-500 shadow-[0_0_30px_rgba(255,0,0,0.3)] bg-red-950/20" : ""
)}>
```

## Winning

When the player reaches 50 correct digits in Normal mode, `endGame(true)` is called:

1. `playSuccessSound()` fires.
2. `canvas-confetti` launches 150 particles spread across 100° from the center of the page, using the app's three signature colors: `['#00D4FF', '#B026FF', '#FF3366']`.
3. A result screen appears showing total digits recalled with options to retry or return to the mode selection menu.

<Tip>
  The game challenges players on digits *after* the decimal point. The `targetDigits` string is `PI_DIGITS.substring(1)`, starting at `1` (from `3.14159...`), so the first digit to type is `1`.
</Tip>
