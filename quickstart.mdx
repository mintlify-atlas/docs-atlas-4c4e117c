## Prerequisites

Before you begin, make sure you have the following installed:

- **Node.js 24** — required by the workspace toolchain
- **pnpm** — the only supported package manager (`npm` and `yarn` are blocked by a preinstall guard)

<Note>
  The root `package.json` enforces pnpm. Running `npm install` or `yarn install` will exit with an error.
</Note>

---

## Run the Pi Day frontend

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone <your-repo-url>
    cd artifacts-monorepo
    ```
  </Step>

  <Step title="Install all workspace dependencies">
    Run this once from the repository root. pnpm installs every package in `artifacts/`, `lib/`, and `scripts/` in one pass.

    ```bash
    pnpm install
    ```
  </Step>

  <Step title="Set required environment variables">
    The Vite config reads `PORT` and `BASE_PATH` at startup and will throw if either is missing.

    ```bash
    export PORT=5173
    export BASE_PATH=/
    ```

    <Tip>
      You can also place these in a `.env` file at the repo root — Vite picks them up automatically during `dev`.
    </Tip>
  </Step>

  <Step title="Start the Pi Day dev server">
    ```bash
    pnpm --filter @workspace/pi-day run dev
    ```

    Vite starts on `http://0.0.0.0:<PORT>`. Open `http://localhost:5173` (or whichever port you set) in your browser.
  </Step>
</Steps>

## Run the API server

<Steps>
  <Step title="Provision a PostgreSQL database">
    The API server connects to PostgreSQL via `@workspace/db`. You need a running Postgres instance and its connection string.
  </Step>

  <Step title="Set the DATABASE_URL environment variable">
    ```bash
    export DATABASE_URL="postgresql://user:password@localhost:5432/mydb"
    ```

    <Warning>
      `DATABASE_URL` is required. Both `@workspace/db` (Drizzle client) and `@workspace/db`'s `drizzle.config.ts` will throw at startup if it is not set.
    </Warning>
  </Step>

  <Step title="Push the database schema">
    Apply the Drizzle schema to your database. In development, use the `push` command instead of migrations:

    ```bash
    pnpm --filter @workspace/db run push
    ```

    If you encounter constraint conflicts during iteration, use the force variant:

    ```bash
    pnpm --filter @workspace/db run push-force
    ```
  </Step>

  <Step title="Start the API dev server">
    ```bash
    pnpm --filter @workspace/api-server run dev
    ```

    The server starts with `tsx` (no compile step) and listens on the port defined by the `PORT` environment variable. The health endpoint is available at `http://localhost:<PORT>/api/healthz`.
  </Step>
</Steps>

## Build for production

To produce production artifacts for all packages that define a `build` script, run from the repo root:

```bash
pnpm run build
```

This runs `typecheck` first (TypeScript composite build across all project references), then recursively calls `build` in each package. The API server compiles to a single CJS bundle at `artifacts/api-server/dist/index.cjs`.

<CodeGroup>

```bash Build everything
pnpm run build
```

```bash Typecheck only
pnpm run typecheck
```

```bash Build API server only
pnpm --filter @workspace/api-server run build
```

</CodeGroup>

## Regenerate API client code

The React Query hooks and Zod schemas in `lib/api-client-react` and `lib/api-zod` are generated from the OpenAPI spec. After editing `lib/api-spec/openapi.yaml`, regenerate them:

```bash
pnpm --filter @workspace/api-spec run codegen
```

<Note>
  The codegen output is written into `lib/api-client-react/src/generated/` and `lib/api-zod/src/generated/`. Commit the generated files alongside your spec changes.
</Note>
