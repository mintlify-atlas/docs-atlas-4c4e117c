title: "API server overview"
description: "Express 5 HTTP server that mounts all routes under the /api base path."
---

The API server lives in `artifacts/api-server` and is the single deployable HTTP service in this monorepo. It is built on [Express 5](https://expressjs.com/) and written in TypeScript. In development it runs directly with `tsx`; for production it is bundled into a single CJS file by esbuild.

## Base URL

All routes are mounted under the `/api` prefix. The server reads the `PORT` environment variable at startup and refuses to start if it is absent or invalid.

## Available endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/healthz` | Returns server health status |

## App setup

`src/app.ts` creates the Express application, attaches middleware, and mounts the router:

```typescript
// artifacts/api-server/src/app.ts
import express, { type Express } from "express";
import cors from "cors";
import router from "./routes";

const app: Express = express();

app.use(cors());
app.use(express.json());
app.use(express.urlencoded({ extended: true }));

app.use("/api", router);

export default app;
```

Middleware applied in order:

1. **CORS** — allows cross-origin requests from any origin (default `cors()` config).
2. **JSON body parsing** — parses `application/json` request bodies.
3. **URL-encoded body parsing** — parses `application/x-www-form-urlencoded` bodies.
4. **Router** — all route handlers mounted at `/api`.

## Entry point

`src/index.ts` reads `PORT`, validates it, and starts the HTTP listener:

```typescript
// artifacts/api-server/src/index.ts
import app from "./app";

const rawPort = process.env["PORT"];

if (!rawPort) {
  throw new Error(
    "PORT environment variable is required but was not provided.",
  );
}

const port = Number(rawPort);

if (Number.isNaN(port) || port <= 0) {
  throw new Error(`Invalid PORT value: "${rawPort}"`);
}

app.listen(port, () => {
  console.log(`Server listening on port ${port}`);
});
```

## Route structure

`src/routes/index.ts` is the root router. Each feature area gets its own sub-router file and is mounted here:

```typescript
// artifacts/api-server/src/routes/index.ts
import { Router, type IRouter } from "express";
import healthRouter from "./health";

const router: IRouter = Router();

router.use(healthRouter);

export default router;
```

## Running the dev server

```bash
pnpm --filter @workspace/api-server run dev
```

This runs `NODE_ENV=development tsx ./src/index.ts`. The `PORT` environment variable must be set beforehand (Replit sets it automatically).

## Dependencies

The server depends on two workspace libraries:

- **`@workspace/api-zod`** — generated Zod schemas used for response validation.
- **`@workspace/db`** — Drizzle ORM client and schema for database access.
